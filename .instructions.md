# Runtipi Custom App Store - Development Instructions

This repository maintains a custom app store for Runtipi. All development must follow the comprehensive guidelines in `.github/copilot-instructions.md`.

## Key Directives

### Version Management (CRITICAL)
When updating any app version:
1. Update `version` in `config.json` to match the new image tag EXACTLY
2. Update image tag in `docker-compose.json` to match
3. **Increment `tipi_version` by 1** in config.json
4. **Update `updated_at` timestamp** to current milliseconds using `Date.now()`

**COMMON MISTAKE:** Only updating version/image without incrementing `tipi_version` or timestamp.

### Port Assignment (CRITICAL)
- All new apps MUST use ports in range **8800-8999**
- **ALWAYS verify availability first** before creating config files:
  ```bash
  grep -rh '"port"' apps/*/config.json | grep -oP ':\s*\K\d+' | sort -n | uniq
  ```
- Then search docker-compose.json files for conflicts
- **REASON:** Port reuse has happened before (e.g., razor-finance + omoide both used 8840)

### Docker Image Tags
**CRITICAL:** Always verify exact tag format from the ACTUAL REGISTRY before committing:
1. Check where official docs say to pull from (Docker Hub, GHCR, lscr.io, etc)
2. Visit the registry package page directly:
   - Docker Hub: `https://hub.docker.com/r/{owner}/{image}/tags`
   - GHCR: `https://github.com/orgs/{owner}/packages/container/{image}/versions`
   - LinuxServer: `https://hub.docker.com/r/linuxserver/{image}/tags`
3. **Common mistake:** `v1.0.0` vs `1.0.0` - only verify from registry, never assume
4. Copy tag EXACTLY as shown in registry

**Real mistakes that happened:**
- Used `1.4.4` when Docker Hub had `v1.4.4` (Recommendarr)
- Used `v2.23.0` when GHCR had `2.23.0` (qbitwebui)

### Form Fields
**NEVER ask for database credentials** - hardcode them in docker-compose.json with simple defaults.

### Volume Mounting
Pattern: `${APP_DATA_DIR}/data/{last-part-of-containerPath}`
- Container `/var/lib/postgresql/data` → Host `${APP_DATA_DIR}/data/data` ✅
- **NOT** `${APP_DATA_DIR}/data/var/lib/postgresql/data` ❌

### File Structure Rules
- Always use `docker-compose.json` (never `.yml` or `.yaml`)
- Never have both `.json` and `.yml` in same app
- Must include: config.json, docker-compose.json, metadata/logo.jpg, metadata/description.md

### Validation
Always run after changes:
```bash
bun test
```

This validates all configs against schema.
